# Static Assets Integration Guide

This guide explains how to work with static assets (CSS, JavaScript, images) in this Laravel project.

## Asset Pipeline Overview

This project uses **Vite** as the asset bundler, which provides:
- Fast hot module replacement (HMR) during development
- Optimized production builds
- Modern JavaScript and CSS support
- Asset versioning for cache busting

## Directory Structure

```
website-laravel/
├── public/                      # Public-facing directory (web root)
│   ├── build/                  # Compiled assets (generated by Vite)
│   ├── images/                 # Static images (manually added)
│   ├── favicon.ico
│   └── robots.txt
├── resources/
│   ├── css/
│   │   └── app.css            # Main CSS file (compiled by Vite)
│   ├── js/
│   │   ├── app.js             # Main JavaScript file (compiled by Vite)
│   │   └── bootstrap.js       # Bootstrap configuration
│   └── views/                  # Blade templates
└── vite.config.js              # Vite configuration
```

## Working with CSS

### Main Stylesheet

The main stylesheet is located at `resources/css/app.css`. This file is automatically compiled by Vite.

**To add custom styles:**

1. Edit `resources/css/app.css` directly:
```css
/* Add your custom styles */
.my-custom-class {
    color: blue;
}
```

2. Or create a new CSS file and import it in `app.css`:
```css
/* In resources/css/app.css */
@import './custom-styles.css';
```

### Using CSS in Blade Views

Include the compiled CSS in your Blade layouts:

```blade
@vite(['resources/css/app.css', 'resources/js/app.js'])
```

This is already included in `resources/views/layouts/app.blade.php`.

## Working with JavaScript

### Main JavaScript File

The main JavaScript file is located at `resources/js/app.js`.

**To add custom JavaScript:**

1. Edit `resources/js/app.js`:
```javascript
// Add your custom code
document.addEventListener('DOMContentLoaded', function() {
    console.log('App loaded');
});
```

2. Or create a new JS file and import it:
```javascript
// In resources/js/app.js
import './custom-script.js';
```

### Using External Libraries

**Via NPM (recommended):**

```bash
npm install library-name
```

Then import in your JavaScript:
```javascript
// In resources/js/app.js
import libraryName from 'library-name';
```

## Working with Images

### Adding Images

1. **Static images** (logos, icons, etc.):
   - Place them in `public/images/`
   - Example: `public/images/logo.png`

2. **User-uploaded images**:
   - Configure Laravel storage
   - Use `php artisan storage:link` to create symlink
   - Upload to `storage/app/public/`

### Referencing Images in Views

**For static images in `public/` directory:**

```blade
<!-- Direct reference -->
<img src="{{ asset('images/logo.png') }}" alt="Logo">

<!-- Or with URL helper -->
<img src="{{ url('images/logo.png') }}" alt="Logo">
```

**For images from storage:**

```blade
<img src="{{ asset('storage/images/photo.jpg') }}" alt="Photo">
```

### Referencing Images in CSS

```css
.hero {
    background-image: url('/images/hero-bg.jpg');
}
```

## Development Workflow

### Starting Development Server

Run the following commands in separate terminals:

**Terminal 1 - Laravel server:**
```bash
php artisan serve
```

**Terminal 2 - Vite dev server (with HMR):**
```bash
npm run dev
```

Now any changes to CSS or JavaScript will be automatically reflected in the browser.

### Building for Production

When ready to deploy:

```bash
npm run build
```

This creates optimized, minified assets in `public/build/`.

## Asset Organization Best Practices

### CSS Organization

```
resources/css/
├── app.css              # Main entry point
├── components/          # Component-specific styles
│   ├── buttons.css
│   ├── forms.css

```

## Advanced: Multiple Entry Points

If you need separate asset bundles for different sections:

**Edit `vite.config.js`:**
```javascript
import { defineConfig } from 'vite';
import laravel from 'laravel-vite-plugin';

export default defineConfig({
    plugins: [
        laravel({
            input: [
                'resources/css/app.css',
                'resources/css/admin.css',  // Admin-specific CSS
                'resources/js/app.js',
                'resources/js/admin.js',     // Admin-specific JS
            ],
            refresh: true,
        }),
    ],
});
```

**Use in Blade:**
```blade
{{-- For regular pages --}}
@vite(['resources/css/app.css', 'resources/js/app.js'])

{{-- For admin pages --}}
@vite(['resources/css/admin.css', 'resources/js/admin.js'])
```

## Adding Fonts

### Web Fonts (Google Fonts, etc.)

Add to your layout head:
```blade
<link rel="preconnect" href="https://fonts.bunny.net">
<link href="https://fonts.bunny.net/css?family=figtree:400,600&display=swap" rel="stylesheet" />
```

### Self-hosted Fonts

1. Place font files in `public/fonts/`
2. Reference in CSS:

```css
@font-face {
    font-family: 'CustomFont';
    src: url('/fonts/CustomFont.woff2') format('woff2'),
         url('/fonts/CustomFont.woff') format('woff');
    font-weight: normal;
    font-style: normal;
}

body {
    font-family: 'CustomFont', sans-serif;
}
```

## Adding Tailwind CSS (Optional)

If you want to use Tailwind CSS:

1. Install dependencies:
```bash
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init -p
```

2. Configure `tailwind.config.js`:
```javascript
export default {
  content: [
    "./resources/**/*.blade.php",
    "./resources/**/*.js",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}
```

3. Add to `resources/css/app.css`:
```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

4. Rebuild assets:
```bash
npm run dev
```

## Troubleshooting

### Assets not loading

1. Make sure Vite is running (`npm run dev`)
2. Check if assets are built (`npm run build`)
3. Clear Laravel cache: `php artisan cache:clear`
4. Clear browser cache

### Changes not reflected

1. Hard refresh browser (Ctrl+Shift+R or Cmd+Shift+R)
2. Restart Vite dev server
3. Check browser console for errors

### Production assets not found

1. Ensure you ran `npm run build`
2. Check `public/build/manifest.json` exists
3. Verify `@vite` directive is used in Blade templates
